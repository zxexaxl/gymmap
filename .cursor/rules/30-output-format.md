# Output Format

Use concise, practical reporting for implementation tasks.

## Recommended Order
1. Understanding
2. Files inspected
3. Cause or approach
4. Changes made
5. Risks / impact
6. Verification steps

## Communication Rules
- Prioritize concrete, file-level explanations over abstract discussion.
- Keep wording short, direct, and operational.
- Include only details needed to review or continue the work.
