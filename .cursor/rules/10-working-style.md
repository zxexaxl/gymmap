# Working Style

## Core Behavior
- Read relevant files before editing.
- Prefer small, targeted changes over broad rewrites.
- Match existing code style, architecture, and implementation patterns.
- Avoid unrelated refactoring during feature or bug tasks.

## Default Implementation Flow
1. Understand current behavior.
2. Identify and inspect affected files.
3. Decide an approach consistent with existing patterns.
4. Implement with the smallest effective change set.
5. Summarize what changed and what should be checked.

## Ambiguity Handling
- If requirements are incomplete, choose the most natural assumption from existing code behavior.
- State assumptions clearly in the final explanation.
